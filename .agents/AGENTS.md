# Project Rules

- **Git Synchronization**: Before performing any development, modification, or testing operations, always synchronize with the latest version of the remote GitHub repository by running `git pull origin main` to ensure that all tasks are based on the latest code.
- **GitHub優先原則**：進行任何開發、修改或測試操作前，必須先從 GitHub 遠端倉庫同步最新代碼（例如執行 `git pull origin main`），確保所有操作均以 GitHub 最新倉庫為準。
- **Auto-Sync to GitHub**: After any local changes (e.g. data regeneration, code modifications), immediately commit and push the changes to GitHub to ensure the remote repository is always up-to-date.
- **即時同步至 GitHub**：本機進行任何修改（如資料重新生成、代碼修改）後，必須立即將變更 commit 並 push 到 GitHub 遠端倉庫，以維持倉庫資料為最新狀態。
