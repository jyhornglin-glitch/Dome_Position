一、專案簡介
本系統（大巨蛋演繹個人定位與跑位查詢系統）係專為慈濟大巨蛋演繹活動開發之個人化定位與跑位路線查詢工具。系統旨在協助大量演繹人員（破千人）於行動端或電腦端快速輸入個人資訊後，查詢其在多個不同隊形（步驟）中的精確物理座標、相鄰人員關係、移動路徑軌跡以及動作提示。

系統具備高互動性的 SVG 網格地圖、東西隊伍對稱鏡像計算、無快取延遲更新、後台權限管理、Excel/CSV 批次匯入以及 PDF 報表下載等實用功能，支持 Fly.io、PythonAnywhere 及 Synology NAS 等多種伺服器環境之部署。

二、系統架構與核心檔案說明
1. 前端展示與互動 (Frontend Layout & SVG Interaction)
   - index.html: 前端主頁面，基於響應式設計（Responsive Design）適配手機與平板，提供搜尋框、隊伍篩選、隊形選擇與詳細資訊展示。
   - style.css: 前端樣式表，採用皇家藍（Royal Blue）及質感暖黃等現代配色，定義格點、路線指引、浮動面板等樣式。
   - app.js: 核心前端腳本，負責動態繪製 SVG 網格、計算定位與 6 個歷程點軌跡、處理行動端 Pinch-to-Zoom 縮放與平移、處理東西隊伍 X 軸鏡像公式（x = -12 - x）。
   - data.js / daydata.js: 靜態 JSON 資料庫，包含所有演繹場次與一般人員的座標與詳細資訊，供前端進行低延遲、無資料庫查閱。
   - action_hints_data.js: 存放各個定點的演繹動作提示與圖片路徑。

2. 後台管理與伺服器 (Backend Administration & Daemon)
   - server.py: 基於 Python 標準庫開發的輕量級 Web 伺服器，免除第三方框架依賴，方便在 Synology NAS 等受限環境直接運行。負責處理登入驗證、檔案讀寫、後台 API 與 CSV 匯出匯入。
   - admin.html: 後台管理介面，包含密碼登入、演出人員名單查詢與連續修改、CSV 資料庫重新匯入等功能。
   - wsgi.py / start.sh: Web 伺服器的 WSGI 入口點與啟動指令碼。

3. 資料處理與自動化腳本 (Data Pipelines)
   - import_csv.py: 將 performers.csv 轉化為前端 data.js 的 Python 處理工具。
   - import_daycsv.py: 將 dayperformers.csv 轉化為前端 daydata.js 的 Python 處理工具，包含場次轉換與資料清理。
   - import_action_hints.py: 解析 Word 動作提示文件（.docx）並提取圖檔與文字。
   - generate_manual.py: 使用 FPDF 生成大巨蛋演繹系統使用手冊的自動化腳本。
