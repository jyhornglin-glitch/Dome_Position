Challenges & Vision

定位難題與系統願景

大型佛教演繹活動（如慈濟大巨蛋法會）往往面臨超大規模演繹人員（破千人）的管理挑戰。物理場地龐大，且隨唱誦文字需不斷變換多達 7 組以上複雜隊形。

千人規模查閱不易 傳統紙本著名冊查閱極易看錯、找尋效率低，現場定位花費大量排練時間。
動態隊形與路徑交錯 人員需在多個定點間穿梭，缺乏起點與中途點路線指引時，排練容易發生碰撞。
對稱演繹座標換算繁雜 舞台區分東西兩側，傳統需要人工分別計算對稱座標，極易產生筆誤。

Front-End Core

互動式舞台網格定位

前端採用互動式 SVG 技術，完整重現大巨蛋舞台平面與座標刻度。所有跑位軌跡採向量繪製，支援無損縮放與高幀率平移。

動態 7 點跑位軌跡 自動連線「起點、前點、目前點、後點」等移動路線，以彩色箭頭指引方位。
行動端手勢縮放 (Pinch-to-Zoom) 支援行動裝置雙指無損縮放與拖曳，可流暢檢視極細微的站位點。
地圖 45 度旋轉 提供「網格旋轉」功能，可將地圖旋轉 45 度，以貼合現場人員的物理視覺朝向。

Data Logic

東西班對稱鏡像與場次

為了減輕後台資料錄入負擔，系統研發了對稱鏡像演算法，並針對演繹活動不同的天次進行動態數據加載。

西班 X 軸自動物理鏡像 採用對稱鏡像公式：x = -12 - x，西班人員無須手動換算座標，一鍵完成鏡像渲染。
多場次與班別過濾機制 首頁提供「今日演出場次」與「東班 / 西班」選擇，自動對齊正確的資料集。
模糊搜尋與自動聯想 支援輸入身分證編號（如 1-49）或姓名模糊搜尋，並自動彈出即時聯想下拉選單。

Integrations & Exports

動作提示與 PDF 下載

將演繹人員所需的動作細節與物理座標卡片高度整合，並支援離線攜帶的紙本檔案匯出，方便在現場無網路環境下核對。

完整動作提示整合 整合 Word 檔中的合唱手勢與步伐指示，於手機端「動作提示」面板一鍵展開並附縮放圖示。
表格定位地圖 PDF 匯出 基於 jsPDF，一鍵生成包含個人所有定點座標、方向、動作說明及地標圖案的 PDF 報表。
A4 現場列印優化 報表自動適配標準 A4 列印格式，適合現場發放予年長演繹人員手持對照。
Administration

後台權限管理與編輯

內建輕量級管理後台，供行政人員動態修正演出人員名單與精準坐標。提供多層安全防禦。

雙角色權限機制 密碼驗證區分兩類權限卡（如一般管理 tzuchi60 與進階系統 tzuchi6060），實現精細控管。
在線座標編輯器 支援行政人員在網頁介面直接點選表格，進行連選修改、在線填寫新座標並即時儲存。
Excel/CSV 批次匯入 後台提供 CSV 資料庫直接匯入，自動重新整理 data.js/daydata.js 並強制驗證 UTF-8 編碼防出錯。

Automation

自動化資料管道與維護

為了加速行政維護與文件同步，系統實作了自動化提取與文件自動生成管道，省去大量人工操作時間。

Word 動作提示自動解析 import_action_hints.py 可解析 Word 文件並自動裁剪圖檔及轉成 JSON，實現手冊文案的一鍵同步。

Git 不同步排除規則 於 AGENTS.md 設定安全同步排除，防止本地端繁瑣的資料庫修改被 Git 同步意外覆蓋。

Summary & Deploy

系統總結與部署環境

本系統以極輕量化的無資料庫架構（No-Database Architecture）進行設計，將高吞吐量的千人查詢完全置於瀏覽器端執行，實現極致的反應速度。

原生 Python Web 伺服器 server.py 基於 Python 標準庫開發，無須安裝額外依賴套件，啟動極為迅速且相容性極高。
多樣化雲端及私有端部署 支援 Docker 容器化、Fly.io 雲端快速發布、PythonAnywhere 以及 Synology NAS 等本地端私有伺服器。
離線前瞻架構 首頁資料靜態加載，即使伺服器瞬時離線，只要載入成功即可完全在手機本機流暢搜尋與使用。
